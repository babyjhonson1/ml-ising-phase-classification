# Классификация фаз модели Изинга методами машинного обучения

Проект демонстрирует, как методы машинного обучения могут распознавать **фазовый переход** во
двумерной модели Изинга. Конфигурации спинов генерируются методом **Метрополиса–Хастингса**, после
чего модели (логистическая регрессия и многослойный перцептрон) обучаются классифицировать снимок
решётки как принадлежащий **упорядоченной** (ферромагнитной) или **неупорядоченной** (парамагнитной)
фазе.

## Краткая теория

### Модель Изинга

Модель Изинга — одна из базовых моделей статистической физики, описывающая магнетизм. На
двумерной квадратной решётке размера `L × L` в каждом узле находится спин, принимающий одно из двух
значений:

$$
s_i \in \{-1, +1\}
$$

Энергия конфигурации задаётся гамильтонианом (без внешнего поля, с взаимодействием только ближайших
соседей):

$$
E = -J \sum_{\langle i, j \rangle} s_i s_j
$$

где `J > 0` — константа обменного взаимодействия (ферромагнитный случай), а сумма берётся по всем
парам соседних узлов. В проекте используются **периодические граничные условия** (решётка
«замыкается» в тор).

Вероятность конфигурации при температуре `T` подчиняется распределению Больцмана:

$$
P(\text{config}) \propto e^{-E / T}
$$

(константа Больцмана `k_B = 1`).

**Фазовый переход.** При низких температурах спины выстраиваются согласованно — система находится в
**упорядоченной (ферромагнитной)** фазе с ненулевой намагниченностью. При высоких температурах
тепловой шум разрушает порядок — система переходит в **неупорядоченную (парамагнитную)** фазу. Переход
происходит при критической температуре Онзагера:

$$
T_c = \frac{2}{\ln(1 + \sqrt{2})} \approx 2.269
$$

Именно этот порог используется для разметки данных: снимки с `T < T_c` помечаются меткой `0`
(упорядоченная фаза), а с `T ≥ T_c` — меткой `1` (неупорядоченная фаза).

### Алгоритм Метрополиса–Хастингса

Напрямую сэмплировать конфигурации из распределения Больцмана невозможно (число состояний
`2^(L·L)` астрономически велико). Поэтому применяется метод Монте-Карло по схеме марковских цепей
(MCMC) — **алгоритм Метрополиса–Хастингса**:

1. Выбрать случайный узел `(i, j)`.
2. Вычислить изменение энергии при перевороте спина:

   $$
   \Delta E = 2 J\, s_{ij} \sum_{\text{соседи}} s_{\text{сосед}}
   $$

3. Принять переворот по правилу:
   - если `ΔE ≤ 0` — переворот принимается всегда (энергия уменьшается);
   - если `ΔE > 0` — переворот принимается с вероятностью `exp(-ΔE / T)`.

Один **sweep** — это `L·L` таких попыток (в среднем по одной на узел). После большого числа
свипов цепь приходит в равновесие, и снимки решётки можно считать выборкой из распределения
Больцмана.

В коде это реализовано так:

- `metropolis_step` — одна попытка переворота;
- `metropolis_sweep` — `L·L` попыток;
- `thermalize` — прогрев (термализация) системы до равновесия перед началом сбора данных (просто много свипов).

Чтобы соседние снимки были как можно менее скоррелированы, между сохранениями делается несколько
свипов (`sampling_interval`).

## Постановка задачи ML

- **Объект:** снимок решётки `16 × 16`, развёрнутый в вектор из 256 признаков (значения спинов `±1`).
- **Метка:** `0` — упорядоченная фаза (`T < T_c`), `1` — неупорядоченная фаза (`T ≥ T_c`).
- **Задача:** бинарная классификация фазы.

Признаки стандартизуются (`StandardScaler`), выборка делится на обучающую и тестовую с сохранением
баланса классов (`stratify=y`).

Сравниваются две модели:

| Модель | Реализация | Архитектура |
|--------|-----------|-------------|
| Логистическая регрессия | `LogisticRegression` | линейный классификатор |
| Многослойный перцептрон | `MLPClassifier` | скрытые слои `(256, 128)` |

## Запуск

Скрипты запускаются из корня проекта как модули (чтобы работали импорты `src.*`).

**1. Сгенерировать датасет** (метод Монте-Карло, может занять время):

```bash
python -m scripts.generate_dataset
```

Результат сохраняется в `data/ising_snapshots.npz` (массивы `X`, `y`, `T`).

Параметры генерации (см. `scripts/generate_dataset.py`):

- `L = 16` — размер решётки;
- `temperatures = linspace(1.0, 4.0, 80)` — 80 значений температуры;
- `samples_per_temperature = 50` — снимков на каждую температуру;
- `thermalization_sweeps = 500` — свипов на прогрев;
- `sampling_interval = 20` — свипов между сохранениями.

**2. Обучить и оценить модели:**

```bash
python -m src.models.logreg
python -m src.models.mlp
```

Каждый скрипт выводит accuracy, F1-score, classification report, матрицу ошибок и температуры
объектов, на которых модель ошиблась.

## Обсуждение

- **Линейная модель плохо справляется.** Логистическая регрессия работает с развёрнутым вектором
  спинов и не «видит» пространственную структуру решётки. Признак, по которому фазы разделимы
  (например, абсолютная намагниченность), нелинейно зависит от входа, поэтому линейный классификатор
  оказывается близок к случайному угадыванию.

- **MLP уверенно различает фазы.** Нелинейные скрытые слои способны выделить из сырой конфигурации
  признаки, коррелирующие с упорядоченностью, и достигают точности ~96%.

- **Ошибки сосредоточены у критической температуры.** Если посмотреть на температуры неверно
  классифицированных объектов, почти все они лежат вблизи `T_c ≈ 2.269`. Это физически ожидаемо: в
  окрестности фазового перехода конфигурации обеих фаз похожи (растут флуктуации и корреляционная
  длина), и сама бинарная разметка по жёсткому порогу становится условной. Вдали от `T_c` MLP
  практически не ошибается.
